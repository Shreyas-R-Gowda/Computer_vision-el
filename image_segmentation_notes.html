1 Fundamentals of Image Segmentation

Definition

Image Segmentation is the process of partitioning an image R into n non-overlapping, exhaustive subregions (R₁, R₂, …, Rₙ) such that each region is internally homogeneous according to some predefined criterion (e.g., intensity, colour, texture).

Formal Conditions on Subregions

  • Completeness: R₁ ∪ R₂ ∪ … ∪ Rₙ = R (every pixel belongs to some region)
  • Disjointness: Rᵢ ∩ Rⱼ = ∅ for i ≠ j (no pixel is in two regions)
  • Connectivity: Each Rᵢ is a connected set
  • Uniformity: Q(Rᵢ) = TRUE for all i (a predicate is satisfied within the region)
  • Non-mergability: Q(Rᵢ ∪ Rⱼ) = FALSE for adjacent regions (adjacent regions are distinct)

Two Fundamental Properties Used

DISCONTINUITY

  • Partition image based on abrupt changes in intensity
  • Basis for edge-detection methods
  • Looks for boundaries between regions

SIMILARITY

  • Partition image based on similar properties
  • Basis for thresholding & region-based methods
  • Groups pixels with common attributes
💡
Intuition:

Think of an image as a puzzle. Discontinuity-based methods find the edges of puzzle pieces. Similarity-based methods group pieces that look the same colour together. Segmentation ultimately answers: which pixels belong together?

2 Image Thresholding

Definition

Thresholding converts a greyscale image into a binary image by comparing each pixel's intensity to a threshold value T. It is the simplest and computationally cheapest segmentation technique.

General Thresholding Formula
g(x, y) = 1  if f(x, y) > T     ← object pixel (foreground)
g(x, y) = 0  if f(x, y) ≤ T     ← background pixel

A. Global Thresholding

A single constant threshold T is applied uniformly across the entire image.

  • Works well when the image has a bimodal histogram (clear object and background peaks)
  • Illumination must be uniform for reliable results
  • Fast and easy to implement

Advantages

  • Computationally simple and fast
  • Requires only one parameter (T)
  • Works great on uniformly lit images

Disadvantages

  • Fails under non-uniform illumination
  • Sensitive to noise in the image
  • Poor on multimodal histograms

B. Variable Thresholding

The threshold value T varies across the image based on local image properties or spatial coordinates.

Type How T is Determined Best Used When
Local / Regional Based on neighbourhood mean (m_xy) and standard deviation (σ_xy) Illumination varies across the scene
Dynamic / Adaptive Based on spatial coordinates (x, y) of the pixel Gradient illumination or shadow effects
Moving Average Rolling mean along scan lines of the image Fast document scanning applications
Local Threshold Formula
Txy = a·σxy + b·mxy
σ_xyLocal standard deviation in neighbourhood of (x, y)
m_xyLocal mean intensity in neighbourhood of (x, y)
a, bTuning constants determined experimentally
Moving Average Update Formula
m(k+1) = m(k) + (1/n) · [zk+1 − zk−n+1]
m(k)Moving average at position k
z_k+1Intensity of incoming pixel
z_k-n+1Intensity of the oldest pixel leaving the window
nWindow size (number of pixels in average)

3 Optimum Global Thresholding — Otsu's Method

Definition

Otsu's method is a statistical thresholding algorithm that automatically determines the optimal threshold k* by maximising the between-class variance of the pixel intensity distribution, thereby minimising the average within-class variance (classification error).

🎯
Intuition:

Otsu asks: "At which intensity level should I split the histogram such that the two resulting groups (background and object) are as different as possible from each other?" The answer — the value that maximises σ_B²(k) — is the optimal threshold.

Step-by-Step Algorithm

⚙️ OTSU'S THRESHOLDING ALGORITHM
  1. Compute the normalised histogram h(i) of the image. Each bin gives p_i = (number of pixels with intensity i) / (total pixels).
  2. Compute cumulative sums P₁(k) = Σ p_i for i = 0 to k. (This is the probability that a pixel belongs to Class 1, intensities 0…k).
  3. Compute cumulative means m(k) = Σ i·p_i for i = 0 to k.
  4. Compute the global mean m_G = Σ i·p_i for i = 0 to L-1 (all levels).
  5. Compute between-class variance σ_B²(k) for every k using the formula below.
  6. Find the optimal threshold k* = argmax{ σ_B²(k) }.
  7. Evaluate separability η = σ_B²(k*) / σ_G² — close to 1 means easy separation, close to 0 means difficult.
Between-Class Variance (Core Formula)
σ_B²(k) = [m_G · P₁(k) − m(k)]²
              ────────────────────────
              P₁(k) · [1 − P₁(k)]
P₁(k)Cumulative probability of class 1 (background, 0 to k)
1−P₁(k)Probability of class 2 (object, k+1 to L-1)
m(k)Cumulative mean of class 1
m_GGlobal mean of the entire image
k*Optimal threshold = argmax{ σ_B²(k) }

Advantages

  • Fully automatic — no manual tuning
  • Statistically optimal for unimodal separation
  • Uses only the 1D histogram (efficient)
  • Widely supported in OpenCV, MATLAB

Disadvantages

  • Assumes bimodal histogram
  • Fails on multi-class or noisy images
  • Sensitive to non-uniform illumination
  • Ignores spatial information
🌍
Real-World Applications

Medical imaging (separating tissue from background), industrial defect detection (PCB inspection), document digitisation, satellite image analysis (land vs. water).

4 Region-Based Segmentation

A. Region Growing

Definition

Region Growing is a bottom-up segmentation technique that starts from one or more seed pixels and iteratively absorbs neighbouring pixels that satisfy a similarity predicate Q (e.g., intensity difference within a threshold).

🌱 REGION GROWING ALGORITHM
  1. Seed Selection: Identify seed points using a priori knowledge (e.g., user input or automatic detection of local extrema).
  2. Predicate Image: For each pixel, check if the similarity predicate Q holds (e.g., |f(x,y) − seed_value| ≤ δ).
  3. Region Expansion: Append all 8-connected neighbours of the seed that satisfy Q to the region.
  4. Iterative Growth: Repeat expansion for newly added pixels until no more qualifying neighbours exist.
  5. Labelling: Assign a unique label to each resulting connected component.
Why Connectivity Matters: Using only 4-connectivity may fragment regions at diagonal boundaries. 8-connectivity (including diagonal neighbours) gives more natural, contiguous regions.

Advantages

  • Can find non-convex, arbitrarily shaped regions
  • Good at detecting specific objects with a priori knowledge
  • Works well for weld defect, tumour detection

Disadvantages

  • Requires manual or semi-automatic seed selection
  • Sensitive to noise (can grow into wrong areas)
  • Computationally expensive for large images

B. Region Splitting and Merging

Definition

Region Splitting and Merging is a top-down approach that recursively subdivides the image into quadrants using a quadtree structure until all sub-regions satisfy a homogeneity predicate, then merges adjacent similar regions.

🌳 SPLIT AND MERGE ALGORITHM
  1. Initialise: Treat the entire image as a single region Rᵢ.
  2. Split: If Q(Rᵢ) = FALSE (region is not homogeneous), split Rᵢ into four equal quadrants (children in the quadtree).
  3. Repeat: Apply step 2 recursively to all sub-regions until every leaf node satisfies Q.
  4. Merge: If two adjacent regions Rⱼ and Rₖ satisfy Q(Rⱼ ∪ Rₖ) = TRUE, merge them into one region.
  5. Stop: Terminate when no further merges are possible.
🌳
Quadtree Structure:

The root node = entire image. Each node either has 0 children (leaf, homogeneous) or exactly 4 children (split further). The depth of the tree determines minimum region size (e.g., 1 pixel at leaf level).

Advantages

  • Does not need seed points
  • Systematic and structured approach
  • Merging step corrects over-segmentation

Disadvantages

  • Block artefacts from forced quadrant splits
  • Boundaries may not align with true edges
  • Sensitive to the choice of predicate Q

5 Morphological Watersheds (Overview)

Definition

The Watershed Algorithm treats image intensity as a topographic surface (height map). It simulates flooding from regional minima and identifies watershed lines — ridges where flood waters from different basins would meet — as segmentation boundaries.

  • Applied to the gradient magnitude image to detect blob boundaries
  • Produces closed, connected contours — a major advantage over edge detection
  • Common problem: over-segmentation due to noise (solved with markers)
  • Real-world use: Cell/nucleus segmentation in biomedical imaging

7 Quick Revision Summary

THRESHOLDING RECAP

  • Global T → one value for whole image
  • Variable T → local neighbourhood properties
  • Otsu → maximises between-class variance σ_B²
  • Moving avg → fast document scanning

REGION METHODS RECAP

  • Growing → seed → expand via 8-connectivity
  • Split & Merge → quadtree → merge siblings
  • Watershed → flood fill → watershed lines
  • All require predicate Q definition

KEY FORMULAS

  • σ_B²(k) = [m_G·P₁ − m(k)]² / P₁·(1−P₁)
  • T_xy = a·σ_xy + b·m_xy
  • m(k+1) = m(k) + (1/n)[z_k+1 − z_k-n+1]

REAL-WORLD APPLICATIONS

  • Medical: Tumour, cell segmentation
  • Industrial: PCB defect, weld inspection
  • Documents: OCR preprocessing
  • Remote sensing: Land-use mapping

Introduction & Segmentation Basics
SPEAKER 1
Good morning, everyone. Today, our team will walk you through two of the most fundamental categories of image segmentation in Computer Vision — Image Thresholding and Region-Based Segmentation. [CLICK — Title Slide]

Let's start by understanding what we mean by image segmentation. At its core, segmentation is the process of dividing an image into regions that share common properties. [CLICK — Slide 2] Mathematically, if our image is R, we want to split it into subregions R₁ through Rₙ such that their union covers the whole image, they don't overlap, and each region is internally similar — satisfying what we call a predicate Q. [PAUSE]

The two fundamental properties that drive all segmentation algorithms are discontinuity and similarity. Discontinuity-based methods find sharp changes — edges. Similarity-based methods, which are our focus today, group pixels that look alike. Think of it this way: if you're trying to identify a white bone in an X-ray, you group the bright pixels together — that's similarity at work. [POINT TO DIAGRAM — Fig 10.1]
SLIDES 03 – 06
Thresholding — Global, Otsu, & Preprocessing
SPEAKER 2
Thank you. Now let me explain the most widely used similarity-based method — Thresholding. [CLICK — Slide 3]

The idea is beautifully simple. We pick a value T. Any pixel brighter than T becomes white — the object — and anything below becomes black — the background. With one constant for the whole image, this is called Global Thresholding. It works perfectly when the histogram has two clear peaks — one for the background and one for the object — with a deep valley between them. [POINT TO DIAGRAM — Fig 10.32]

But how do we pick the best T automatically? That's where Otsu's Method comes in. [CLICK — Slide 4] Instead of guessing, Otsu treats it as a statistical decision problem. It evaluates every possible threshold value and picks the one that maximises the between-class variance — σ_B squared of k. Intuitively, we want the two groups — background and object — to be as different from each other as possible.

[CLICK — Slide 5] The algorithm steps are: first, compute the normalised histogram. Then calculate cumulative sums P₁(k) and cumulative means m(k). Use these to compute σ_B²(k) for each k, and select k-star — the k that gives the highest variance. That k-star is our optimal threshold. No manual tuning required.

[CLICK — Slide 6] One important concern — what if the image is noisy? Noise spreads the histogram peaks, making the valley between them disappear, so no threshold works well. The solution is to first smooth the image with a Gaussian filter before thresholding. Alternatively, we can build an edge-enhanced histogram by only considering pixels near edges — those pixels have the most discriminative information. [POINT TO DIAGRAM — Fig 10.37] As you can see, the smoothed image gives a cleaner histogram with a more reliable threshold.
SLIDES 07 – 08
Adaptive Thresholding & Region Growing
SPEAKER 3
Thank you. Now, global thresholding has one major weakness — it assumes uniform lighting. What happens when you photograph a document under a lamp that only illuminates the centre? One threshold value will lose either the bright or the dark parts. [CLICK — Slide 7]

This is where Variable or Adaptive Thresholding comes in. Instead of one T for the whole image, we compute a local threshold T_xy at every pixel (x,y) based on its neighbourhood statistics: T_xy equals a times the local standard deviation plus b times the local mean. The constants a and b are tuned for the specific image type. The result is a threshold map that follows the lighting changes across the image.

For applications like document scanning — think CamScanner on your phone — we use a Moving Average approach. As we scan across rows, we maintain a running mean using the formula: m(k+1) equals m(k) plus one-over-n times [z_k+1 minus z_k-minus-n+1]. This efficiently subtracts the oldest pixel from the window and adds the newest, keeping computation fast. [POINT TO DIAGRAM — Fig 10.44] Notice how global thresholding loses the text in dark regions, but the moving average threshold extracts all the text cleanly.

[CLICK — Slide 8] Now we move from thresholding to Region-Based methods — starting with Region Growing. The idea here is to start from a seed point — a pixel we know belongs to our target object — and expand outward by adding any 8-connected neighbour that satisfies our similarity predicate Q. For example, if we're looking for a weld defect, the seed is a pixel in the defect area, and Q might say: include any pixel within 10 intensity units of the current region's mean. [POINT TO DIAGRAM — Fig 10.46]

We keep expanding until no new pixels qualify. The result is a precisely bounded region corresponding to just the defect — no more, no less. This method is powerful when we have prior knowledge about where the object is, but it requires careful seed selection and a well-defined threshold delta.
SLIDES 09 – 12
Split-Merge, Watershed & Conclusion
SPEAKER 4
Thank you. Our final two techniques take a more structural approach. [CLICK — Slide 9] Region Splitting and Merging works top-down. We start with the whole image as one region. If the predicate Q is false — meaning the region is not homogeneous — we split it into four equal quadrants. We keep splitting recursively until every piece is homogeneous. [PAUSE]

Then comes the merging step. Any two adjacent regions that together satisfy Q are merged into one. We stop when no more merges are possible. The data structure behind this is called a Quadtree — the root is the full image, every internal node has exactly four children, and the leaves are homogeneous blocks. [CLICK — Slide 10] [POINT TO DIAGRAM — Fig 10.47] As you can see, the quadtree naturally separates the dense centre from the sparse background, even for a complex astronomical image like the Cygnus Loop. The beauty here is that no seeds are needed — the algorithm discovers regions entirely on its own.

[CLICK — Slide 11] Finally, let's talk briefly about the Watershed Algorithm. Imagine the image gradient as a landscape — high gradient values are mountains, low values are valleys. We flood this landscape from every valley simultaneously. When two floods from different valleys meet, we build a dam — that dam is the watershed boundary. Applied to the gradient magnitude image, this gives us closed, connected boundaries around every distinct object. The main challenge is over-segmentation from tiny noise valleys, which is solved by using markers — pre-defined seeds — to control where flooding starts. [POINT TO DIAGRAM — Fig 10.59]

[CLICK — Slide 12] To summarise: Use Otsu for fast, automatic binary segmentation. Use Adaptive thresholding when lighting is uneven. Use Region Growing when you know where your object is. Use Split-and-Merge for structured automatic segmentation. And use Watershed when you need closed boundaries for touching objects like cells. The right technique depends on your image characteristics and the application at hand. Thank you very much!

C Intuitive Explanations for Complex Concepts

💡
Why does Otsu maximise between-class variance?

If two classes (background & object) have very different means, their between-class variance is high. Maximising this is equivalent to minimising the within-class variance (how spread out each class is internally). A clean threshold makes each class tight and well-separated — minimum classification error.

💡
Why use 8-connectivity in Region Growing?

4-connectivity only considers up/down/left/right neighbours. A diagonal line of pixels would be treated as disconnected. 8-connectivity includes diagonals, so regions grow naturally in all directions and don't get artificially fragmented at 45° boundaries.

💡
Moving Average — Why subtract the old pixel?

Instead of re-summing n pixels from scratch for every new position (O(n) per pixel), we maintain a running sum. Adding the new pixel and subtracting the one that just left the window gives us the updated average in O(1) per pixel — dramatically faster for scanning long document rows.

⚠️
Common Exam Mistakes

1) Confusing cumulative mean m(k) with global mean m_G — m(k) is up to level k, m_G is the overall mean. 2) Forgetting that region growing requires 8-connectivity for continuous regions. 3) Stating Otsu works for multimodal histograms — it's designed for bimodal; multi-Otsu is needed for more classes.
